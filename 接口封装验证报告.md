## 🔍 接口规范检查

### ✅ 请求封装
```typescript
// src/utils/request.ts
- 支持 GET/POST/PUT/DELETE/DOWNLOAD
- 统一对象参数格式: { url, params, data }
- 自动添加 Token
- 统一错误处理
- 401 自动跳转登录
```

### ✅ 类型定义
所有 API 都包含完整的 TypeScript 类型定义：
- `VO` - 视图对象（View Object）
- `ReqVO` - 请求对象（Request VO）
- `PageReqVO` - 分页请求对象

### ✅ 命名规范
- 查询分页: `get{Module}Page`
- 查询详情: `get{Module}`
- 新增: `create{Module}`
- 修改: `update{Module}`
- 删除: `delete{Module}`
- 导出: `export{Module}`
- 审核通过: `approve{Module}`
- 审核拒绝: `reject{Module}`

### ✅ 接口路径规范
```
/{module}/{submodule}/{action}

示例:
- /system/user/page
- /goods/goods/create
- /order/order/deliver
- /pharmacy/prescription/approve
```
