## 📝 登录流程

1. 用户在前端输入用户名和密码
2. 前端调用 `/system/auth/login` 接口
3. 后端验证用户名和密码（BCrypt）
4. 验证成功返回 `accessToken` 和 `refreshToken`
5. 前端保存 token 到 localStorage
6. 前端保存用户信息到 Pinia store
7. 跳转到首页 `/home`

---

## 🔐 Token 管理

### Token 存储
- **位置**: localStorage
- **Key**: `token`
- **Value**: `accessToken` (从登录接口返回)

### Token 使用
- 所有后续请求自动在请求头添加:
  ```
  Authorization: Bearer {accessToken}
  ```

### Token 过期处理
- 后端返回 401 状态码
- 前端自动清除 token
- 弹窗提示用户重新登录
- 跳转到登录页
